1 . What are difference between map , filter , forEach and find array methods ?

Answer :

( i ) map method- .map() executes the same code on every element in an array and returns a new array with the updated elements.

Example :

const arr = [1, 2, 3, 4];
const newArr = arr.map((item) => {
return item * 2 ;
});

( ii ) filter method- .filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.

Example :

const arr = [1, 2, 3, 4];
const newArr = arr.filter((item) => {
return item > 2;
});

( iii ) forEach method- .forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.

Example :

const arr = [1, 2, 3, 4];
arr.forEach((item) => {
console.log(item * 2);
});

( iv ) map method- .find() method returns the value of the first element in the array that satisfies the provided function. The provided function takes in the current element and returns a boolean value (true or false). If the function returns true for an element, find() returns that element. Otherwise, it returns undefined.

Example :

const arr = [1, 2, 3, 4];
const result = arr.find((item) => {
return item > 2;
});

2 . What are the difference between local storage , session storage and cookies ?

Answer :

( i ) local storage- local storage is a mechanism that allows web pages to store key/value pairs in the user's web browser. It can only be accessed by client-side scripts.Local Storage has a larger storage capacity typically up to 5-10MB. The data is retained even after the browser is closed.

( ii ) session storage- session storage is similar to Local Storage, but it is used to store data that is specific to a particular session, which ends when the user closes the browser tab or window. The data stored in Session Storage is only accessible to the client-side scripts.

( iii ) cookies- cookies are small pieces of data that are stored on the user's computer by the web browser. They are primarily used for session management, personalization, and tracking. Cookies can be accessed by both the server and the client-side scripts.

3 . What is event loop? how it is work ?

Answer :

The event loop is a critical part of the JavaScript runtime environment, responsible for handling asynchronous operations in JavaScript.

When a function is called that performs an asynchronous operation , the function is removed from the normal call stack and placed in a queue of deferred tasks. The event loop continues to execute tasks from the normal call stack until it is empty, and then it checks the queue of deferred tasks. If there are any tasks in the queue, the event loop will execute them one by one. As each task is executed, it may perform further asynchronous operations, which are added to the queue of deferred tasks. This process continues until all the tasks in the queue have been executed, and the event loop is idle again.
